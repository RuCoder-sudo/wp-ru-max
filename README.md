## Что такое WP Ru-max

**WP Ru-max** — это WordPress-плагин, который связывает ваш сайт с мессенджером [MAX (max.ru)](https://max.ru). Один плагин = три модуля:

- **Автопубликация** — новые записи и страницы автоматически уходят в канал MAX
- **Личные уведомления** — все письма WordPress (заказы WooCommerce, заявки с форм) дублируются в личный чат с ботом
- **Чат-виджет** — плавающая кнопка MAX с приветствием, звуком и попапом удержания

## Новое в версии 1.0.36

| Что добавлено | Описание |
|---|---|
| **Поддержка bbPress форумов** | Плагин корректно работает с топиками bbPress (тип записи `topic`) — автоматически находит изображение даже без миниатюры. |
| **Автовыбор изображений** | 3 источника по приоритету: миниатюра записи → первый `<img>` из контента → прикреплённый файл. |
| **Логирование источника** | В журнале теперь видно откуда взята картинка — для удобной диагностики. |
| **Совместимость** | WordPress 5.8 – 6.7, PHP 7.4 – 8.3, bbPress 2.6+. |

## Новое в версии 1.0.32

| Что добавлено | Описание |
|---|---|
| **WordPress Multisite** | Плагин корректно работает на всех подсайтах сети. Заголовок `Network: true` — поддержка сетевой активации. |
| **Сетевая лицензия** | Страница «Сеть → Ru-max (Сеть)» для суперадминистратора. Одна лицензия покрывает все подсайты сети. |
| **Поддомены** | Лицензия на `example.com` автоматически покрывает `shop.example.com`, `blog.example.com` и любые другие поддомены того же домена. |
| **Тумблер Multisite** | В «Дополнительных настройках» — переключатель «Включить поддержку сети и поддоменов». |
| **Реорганизация assets** | CSS-файлы перенесены в `assets/css/`, JS-файлы в `assets/js/`. Все пути в PHP обновлены. |
| **Безопасность лицензии** | `scope=subdomain` строго ограничен поддоменами того же домена. Чужие домены не покрываются. |
| **Совместимость** | WordPress 5.8 – 6.7, PHP 7.4 – 8.3. |
