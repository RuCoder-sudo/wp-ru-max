# Changelog for WP Ru-max

Все значимые изменения в этом проекте будут задокументированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.31] - 2026-05-23

### Added
- **Кнопка «Поделиться в MAX»** (Дополнительные настройки) — тумблер включения/выключения кнопки «Поделиться в MAX» в конце каждой статьи и страницы. Ссылка формируется автоматически с адресом и заголовком материала (`max.ru/share?url=…`). Кнопка стилизована в фирменном синем цвете MAX.
- **MAX-авторизация (OAuth)** (Дополнительные настройки) — тумблер включения входа на сайт через аккаунт мессенджера MAX. При включении кнопка «Войти через MAX» появляется на странице входа WordPress, странице регистрации, в форме входа WooCommerce и на странице оформления заказа WooCommerce. Пользователи автоматически привязываются по MAX ID или email; при первом входе создаётся аккаунт WordPress с ролью «Подписчик» и автоматически заполняется имя и аватар.
- В разделе MAX-авторизации добавлены поля **App ID** и **App Secret** (получаются на max.ru/partner) и отображается Redirect URI для настройки OAuth-приложения на стороне MAX.

### Changed
- Версия плагина обновлена до 1.0.31 во всех файлах.

## [1.0.30] - 2026-05-20

### Added
- В «Личных уведомлениях» новый раздел «Правила отправки» с двумя независимыми переключателями: уведомления об обновлении плагинов/ядра WordPress и уведомления критических ошибок сайта.
- При обновлении плагинов WordPress плагин автоматически отправляет сообщение в MAX со списком обновлённых плагинов (или версию ядра).
- При фатальной PHP-ошибке плагин отправляет уведомление в MAX (не чаще 1 раза в 5 минут, антиспам через transient).

### Changed
- Блок «Отправка push-уведомлений в MAX» перенесён под «Канал(ы)» — теперь располагается сразу под выбором каналов для удобства.

## [1.0.29] - 2026-04-15

### Added
- Раздел «Отправка push-уведомлений в MAX» во вкладке «Отправка публикаций» — отправка произвольного текста прямо в канал/группу MAX одним кликом без создания записи.
- Поддержка WP Mail SMTP (и аналогов FluentSMTP, Postman SMTP) — перехват `wp_mail()` теперь выполняется на приоритете фильтра `5`, что гарантирует работу до SMTP-подмены сторонними плагинами.
- Маскировка токена бота в логах — показываются только первые и последние символы, безопасно для экспорта логов.
- Rate-limiting для push-отправок — не более 10 отправок в минуту на пользователя (защита от случайного спама).
- Кэширование ответа API `/me` через WordPress transient на 5 минут — уменьшает количество внешних запросов при частой проверке подключения.

### Changed
- Обновлён «Tested up to» до WordPress 7.0.

## [1.0.28] - 2026-03-10

### Added
- Новая анимация «Левитация» для кнопки чат-виджета — плавное парение вверх-вниз с динамической тенью.

### Changed
- Настройка «Автоотправка по умолчанию» теперь отображается как тумблер в стиле Gutenberg вместо обычного чекбокса.

## [1.0.27] - 2026-02-20

### Added
- Поддержка Jetpack Contact Form — email-адреса в теле уведомлений автоматически экранируются (`@` → `[at]`), MAX больше не делает их кликабельными ссылками `mailto`.
- Глобальная настройка «Автоотправка по умолчанию» в разделе «Отправка публикаций» — если включена, тумблер «Автоотправка в MAX» будет ВКЛ для всех новых записей, у которых ещё не задано явное значение.

### Changed
- Тумблер в редакторе (Gutenberg и Classic Editor) теперь отражает глобальный «По умолчанию» при открытии новой записи.
- Длинные списки категорий и тегов в разделе «Отправка публикаций» теперь сворачиваются — показываются первые 8, остальные скрываются под кнопкой «Показать все N». Выбранные элементы всегда видны.
- Описание плагина обновлено: Jetpack добавлен в список поддерживаемых источников уведомлений.

## [1.0.26] - 2026-01-15

### Added
- Поддержка Classic Editor (TinyMCE) — блок «Отправить в MAX» теперь отображается в панели «Опубликовать». Включает переключатель автоотправки и кнопку «Отправить в MAX вручную» — аналогично интерфейсу Gutenberg.

### Changed
- Переключатель автоотправки в Classic Editor меняет надпись ВКЛ/ВЫКЛ динамически без перезагрузки страницы.

## [1.0.25] - 2025-12-20

### Added
- Индикатор статуса подключения (зелёный / красный кружок) на главной вкладке настроек — обновляется при нажатии «Проверить подключение».
- Отложенная отправка публикаций — новая настройка «Задержка отправки» (0 / 30 сек / 1 / 2 / 5 мин) через WP-Cron; исключает гонку условий между событием сохранения и финальной обработкой изображения.
- Повторные попытки при ошибке API — настраиваемое количество попыток (0–5) и интервал между ними (1–30 сек).
- Ограничение размера изображения — новая настройка «Максимальный размер изображения (МБ)»; изображения тяжелее лимита автоматически пропускаются (отправляется только текст).
- Фильтр по категориям и тегам — можно ограничить автоотправку только записями из выбранных категорий или с определёнными тегами (пустое поле = отправлять всё).

### Fixed
- Гонка условий между отправкой поста и обработкой изображения — отправка теперь производится через механизм отложенного cron-события.

### Changed
- Интервал фоновой перепроверки лицензии увеличен с 1 часа до 160 дней (13 824 000 секунд).

## [1.0.24] - 2026-04-29

### Fixed
- (Критично) Тумблер «Автоотправка в MAX» в Гутенберге сбрасывался в ВЫКЛ после сохранения. Решение — собственный REST-эндпоинт `/wp-json/wp-ru-max/v1/skip/{post_id}` плюс AJAX-fallback.
- Мета-ключ переименован с `_wp_ru_max_skip` на `wp_ru_max_skip` (без подчёркивания) — устранена проблема «protected meta» в WordPress REST API.
- Добавлена одноразовая миграция значений из старого ключа.

### Changed
- Поведение по умолчанию изменено: автоотправка в MAX теперь ВЫКЛЮЧЕНА для всех новых статей.
